ConstraintLayout总结

Jun 28, 2018

介绍

一个可以将降低层次深度做到极致的布局，同时还提供了多种复杂动画效果

配置预备

基本约束定位常用属性

粗鲁说法： 即 layout_constraint（单词） 描述的是 当前视图的边缘位置， _to（单词）Of 描述的是目标视图的边缘对象

对应 Relatelayout的属性 layout_toRightOf layout_toLeftOf layout_alignParent

长宽相关属性

ConstraintLayout 不支持 match_parent，如果想要类似Relatelayout那样自然铺满或者等比铺满当前屏幕需要适当使用

否则会超出布局不能完全显示与屏幕中

当前视图等比例显示长宽

相关属性 app:layout_constraintDimensionRatio="15:4"

线性链约束相关属性

app:layout_constraintVertical_chainStyle
app:layout_constraintHorizontal_chainStyle


可选值：

  • spread
  • packed
  • spread_inside

默认值为 ： spread

spread + 互相约束下：

  • 情况1 :如果全部使用match_constraint 即 0dp ，就会等比平分长度铺满屏幕，也可以同时使用app:layout_constraintHorizontal_weight等要求一定的比例进行等比平分铺满

  • 情况2：如果所有width!=0，而且总长度少于屏幕长度，会平分剩余空白作为各个控件的间距，屏幕边缘与控件也参与瓜分间距（即3个控件会被瓜分出4个间距），否则会被按照顺序挤出屏幕

  • 情况3：如果其中部分width!=0，而且部分为0，width=0 的控件会等比平分长度铺满剩余的位置

spread_inside + 互相约束下：

  • 情况1：如果全部width!=0, 最两边边缘的控件会向边缘对齐，如果总width小于屏幕长度则会把剩余控件平分作为间距铺放每一个控件，否则会被其中最长的挤压（不确定）

  • 情况2：如果部分width!=0 部分为0， 而且总width小于屏幕长度则会把剩余长度（屏幕长度-所有width）平分长度给width=0的控件铺满，否则会比其中最长的挤压（不确定）

packed + 互相约束下：

  • 情况1：如果全部width!=0, 总长度少于屏幕则会全部控件紧贴一起然后居中显示，剩余长度为平分两边与边缘的间距；否则会被其中最长的挤压（不确定）

  • 情况2：如果部分width!=0,部分为0，若总width少于屏幕长度则会把剩余长度分平分给非0的控件作为长度铺满屏幕，否则会被其中最长的挤压（不确定）

不确定字眼是因为还没清楚期其挤压逻辑，初步尝试规则是如果其最长一控件width小于屏幕则优先显示，大于屏幕则不管位置如何都会优先显示最长控件铺满屏幕，日后待解决。
