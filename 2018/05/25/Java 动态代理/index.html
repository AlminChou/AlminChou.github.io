Java 动态代理源码解析

May 25, 2018

Java 动态代理:

AOP的实现机制就是利用动态代理,通过预编译方式和运行期动态代理实现程序功能的统一维护，AOP 是 面向切面编程, 是OOP的延续、函数式编程的一种衍生范型

静态代理缺点：

一个代理只能对应一个服务对象，若有多个服务对象则需要重新对每一个对象进行实现一个静态代理对象，工作代码量比较大，而且容易导致代码复杂。

原理：

不需要显式实现目标对象的每一个相同的接口，而将这一部分工作完全交给jvm去执行; 然后通过调用代理类中的handler对象的invoke进行调用目标对象的真是方法

实现方法：

通过实现InvocationHandler接口调用处理器类对象newProxyInstance（），复写InvocationHandler接口的invoke（）从而调用真实方法，可以在调用真是方法前后或者过程中做一些动作。

如何生成动态代理类：

首先调用 Proxy.newProxyInstance(ClassLoader loader, Class<?>[]interfaces, InvocationHandler h)
传入以下三个参数

参数1：指定产生代理对象的类加载器，需要将其指定为和目标对象同一个类加载器
参数2：指定代理对象需要实现的接口类数组，即传入的数组的所有接口类的所有方法都被该代理对象默认实现了，这样就能调用这组接口中的方法

参数3：指定InvocationHandler对象。即动态代理对象在调用方法时，会关联到哪个InvocationHandler对象
