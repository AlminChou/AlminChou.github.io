Android O 适配

Oct 4, 2018

主要更改行为

  • 对M动态权限进一步优化
  • 通知栏继续更改
  • 桌面图标变化
  • 去除了“允许未知来源”选项，安装APK功能更改
  • 后台执行限制
  • 悬浮窗适配

targetSdkVersion为 26（8.0） or 27（8.1)

动态权限的改变

Android N -> 申请小组其中一个权限则获取了整组权限

Android O -> 只获取了当前申请的一个权限，不再直接获取了整组权限，需要继续申请组内其他权限

适配方案：

状况1：
    前提：如果原来就是按需类型申请
    方案：不需要更改

状况2：
    前提：如果原来是只申请了一个然后使用了其他权限
    方案：可以直接一次过申请整组权限，因为同组权限申请时候只会显示一个窗口不会重复。

允许未知来源 权限

Android O的允许未知来源开关被移除了，换成了每个应用独自的未知来源管理列表。
所以如果app包含了安装apk的功能，需要先满足以下适配工作：

1.先在AndroidManifest文件中添加安装未知来源应用的权限:
<uses-permission android:name="android.permission.REQUEST_INSTALL_PACKAGES"/>
这样系统会自动询问用户完成授权，并不需要主动申请操作


2.先判断是否拥有该权限
getPackageManager().canRequestPackageInstalls();

true : 应用有安装未知来源应用的权限，可以直接执行安装应用的操作
false :应用没有安装未知来源应用的权限，无法安装应用

这里提供的思路是：
if(true){
    // install apk
}else{
    // 提示并且引导用户进去设置页面进行打开权限再继续操作
}
