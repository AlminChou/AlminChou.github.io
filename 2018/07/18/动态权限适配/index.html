6.0动态权限适配总结

Jul 18, 2018

简述

自从Android M开始，官方为提高用户的私人信息安全，将权限分为两大类：普通权限 ，危险权限。

权限分类

  • 普通权限 : 可以直接书写在Mainfest.xml中进行获取权限。

  • 危险权限 : 需要同时书写在Mainfest.xml中再动态在代码中进行申请才能获取权限。(以下9组)

总结:

  • 所有的权限仍然在manifest中静态声明, 普通权限的在安装的时候自动授权, 而危险权限需要应用明确地请求用户授权

  • 危险权限只需要选择每一组其中一个申请后就可以获取整组的权限

适配场景

只有满足targetSdkVersion 大于等于 23 时候才会采用新的动态权限机制. 其他情况, 跟之前一样, 在安装和升级应用的时候就授权了所有的权限.

涉及Api

  • PermissionChecker.checkSelfPermission（）
  • ActivityCompat.shouldShowRequestPermissionRationale(@NonNull Activity activity,
    @NonNull String permission)

  • Fragment.shouldShowRequestPermissionRationale(@NonNull String permission)

  • ActivityCompat.requestPermissions(final @NonNull Activity activity,

    final @NonNull String[] permissions, final int requestCode)

  • Fragment.requestPermissions(@NonNull String[] permissions, int requestCode)

  • public void onRequestPermissionsResult(int requestCode, String[] permissions, int[] grantResults)

注意区分当前请求权限的载体是Activity还是Fragment，使用各自对应的api，Activity使用ActivityCompat的api，而Fragment使用自身的api；最后各自载体也需要重写自身的
@Override public void onRequestPermissionsResult(int requestCode, String[] permissions, int[] grantResults) 进行回调处理。

申请步骤

1.要把需要申请的权限先注册到Mainfest.xml文件中

2.调用PermissionChecker.checkSelfPermission()检测当前需要申请的权限是否已经授权过，是则直接返回true可以直接使用权限，否则需要进行下一步申请

2.5（可选）在3调用之前可以先调用shouldShowRequestPermissionRationale验证是否需要为当前申请的权限对用户进行解释再申请，返回true为需要，false则不需要，可以再此处根据返回值显示一些ui信息反馈给用户

3.当前权限没有被申请允许过时候，调用requestPermission进行初步申请

4.覆盖实现载体的onRequestPermissionsResult进行对申请结果的处理，grantResults的取值为 PackageManager.PERMISSION_GRANTED 与 PackageManager.PERMISSION_DENIED ，即授权与拒绝

5.grantResults 取值为 PackageManager.PERMISSION_DENIED时候需要再调用一次shouldShowRequestPermissionRationale查看是否用户点击了不在弹出询问且拒绝，false即不再询问，应该为适配各种手机进行不一样的操作处理，例如带到设置页面请用户自己打开权限等

注意事项

1.一定要书写到xml  并且代码中申请的为同一个权限引用，不然`checkSelfPermission`永远false，也不会弹出询问窗口。

2.记得处理用户点击不再询问且拒绝后的操作

3.部分手机例如小米，shouldShowRequestPermissionRationale会永远false，而且checkPermission需要用到AppOpsManager类

4.可以同时申请多个权限，requestPermission接受的对象是数组，询问窗口会一个个接着显示，当全部询问结束后才会回调result而且结果是数组顺序形式返回

5.当target大于等于23时候，不做权限获取适配会在触发相对应涉及的功能后会直接crash

6.第一次请求权限时，用户点击了“拒绝”，第二次再请求该权限时，对话框将出现“不再询问”复选框，如果用户勾选了“不再询问”并点击了“拒绝”，则之后再请求此权限组时将不弹框，而是直接返回“拒绝”的结果。
